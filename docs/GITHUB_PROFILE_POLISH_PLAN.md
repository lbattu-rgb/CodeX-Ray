# GitHub Profile Polish Plan

## Immediate Actions

1. Set a clear bio
2. Add website / LinkedIn
3. Pick 6 pinned repos intentionally
4. Clean descriptions for every public repo
5. Remove or hide weak repos

## Pinned Repo Mix Recommendation

1. `AlgoScope`
2. Best ML project
3. Best full-stack app
4. Best data or visualization project
5. Best competition or hackathon project
6. Best technically clean repo

## Repo Description Formula

Use:

`Short noun phrase + what makes it special`

Examples:

- `Algorithm intelligence platform for visualizing and predicting code behavior`
- `Explainable ML dashboard for uncertainty-aware prediction`
- `Full-stack developer tool for interactive code analysis`

## What To Remove From Public View

Hide or archive repos that are:

- unfinished with no README
- duplicates
- tutorial clones with minimal changes
- messy experimental repos with poor naming

## Ideal First Impression

Someone should be able to tell in under 10 seconds:

- what you build
- how serious you are
- which project is your flagship
