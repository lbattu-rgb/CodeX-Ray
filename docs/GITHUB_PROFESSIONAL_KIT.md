# GitHub Professional Kit

This file gives you a clean, professional GitHub setup you can apply today.

## 1. Profile Checklist

Use this as your baseline:

- Professional profile photo
- Clear name
- Short bio with role + focus
- Location and school/company if relevant
- Portfolio or LinkedIn link
- 6 pinned repositories max
- Consistent README quality across top projects
- No messy repo names like `test123`, `new-project-final`, or `random`

## 2. Strong Bio Options

Pick one and customize it.

### Option A

`Building ML, systems, and developer tools with a focus on explainability, visualization, and real-world impact.`

### Option B

`CS student building ambitious AI, systems, and full-stack projects with strong product thinking.`

### Option C

`I build machine learning, developer tooling, and interactive software that makes complex systems easier to understand.`

## 3. What To Pin

Pin projects that show range, not just quantity.

Best mix:

- 1 flagship project
- 1 strong ML project
- 1 full-stack product
- 1 systems or algorithms project
- 1 polished hackathon/competition project
- 1 repo that shows code quality and documentation discipline

## 4. What Makes a Repo Look Professional

Every serious repo should have:

- A strong README
- Clear setup instructions
- Screenshots or demo GIFs if UI exists
- Project structure section
- Tech stack section
- Features section
- Roadmap or future improvements
- License

## 5. README Structure

Use this exact order:

1. Project name
2. One-line value proposition
3. Demo or screenshot
4. Features
5. Tech stack
6. Architecture overview
7. Local setup
8. Usage
9. Future work

## 6. Repo Naming Rules

Use names like:

- `algoscope`
- `uncertainty-aware-ai`
- `gene-visual-intelligence`
- `algorithm-xray`

Avoid:

- `project1`
- `finalfinal`
- `myapp`
- `coolstuff`

## 7. Commit Style

Use clean commit messages:

- `feat: add Monaco-based code editor`
- `fix: sync trace slider with active editor line`
- `refactor: split analysis pipeline into engine modules`
- `docs: add architecture notes and setup guide`

## 8. Profile README Strategy

Your profile README should not be a life story.

It should do 4 things:

- say who you are
- show what you build
- highlight your strongest projects
- make it easy to contact or learn more

## 9. Fast Cleanup Wins

Do these first:

- Archive or make private low-quality repos
- Rename messy repos
- Add descriptions to public repos
- Add topics/tags to important repos
- Make pinned repos intentional
- Add demo screenshots to best repos

## 10. Best Topics To Add

Examples:

- `machine-learning`
- `python`
- `react`
- `fastapi`
- `data-visualization`
- `developer-tools`
- `algorithms`
- `explainable-ai`
- `full-stack`

## 11. Professional Standard

A professional GitHub should feel like:

- thoughtful
- deliberate
- easy to scan
- technically strong
- consistent

If a recruiter, judge, or engineer lands on your profile, they should instantly understand what kind of builder you are.
